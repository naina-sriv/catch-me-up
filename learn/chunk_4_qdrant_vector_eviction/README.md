### 👶 The Concept (Explain it with Easy Examples)
**PostgreSQL (Normal DB) vs. Qdrant (Vector DB)**
In a normal CRUD app, if you search a database for the word "budget", it looks for exact letters. If a transcript says "financial planning", a normal database ignores it. It searches by *keywords*, not by *meaning*.

A **Vector Database** (like Qdrant) searches by meaning. We pass our sentences to an AI, and the AI translates them into **Vectors** (a massive array of GPS-style coordinates). Sentences that mean similar things (like "budget" and "financial planning") get placed physically close together on this mathematical map. When you search, Qdrant just measures the distance between coordinates to find matches!

**Why do we need this? (Eviction Cascades)**
RAM is expensive and limited. If a meeting goes on for 5 hours, Redis will run out of RAM and crash (Out-Of-Memory). 
To fix this, we need a "Garbage Collector" (Eviction Cascade). Every 30 minutes, a background worker sweeps up the oldest messages, converts them into **Vectors**, and dumps them permanently into Qdrant. Then, it deletes the old messages from Redis to free up RAM!

### 🐣 The Simple Version (How to build it first)
```python
# Pseudo-code to explain the concept step-by-step
async def simple_evict(meeting_id: str, oldest_time: float):
    
    timeline_key = f"meeting:{meeting_id}"
    
    # 1. READ OLD DATA: 
    # We ask Redis: "Give me every message from time 0 up to the 'oldest_time' limit."
    old_messages = await redis_client.zrangebyscore(timeline_key, 0, oldest_time)
    
    if not old_messages:
        return # Nothing to clean up
        
    # 2. SAVE IT TO PERMANENT STORAGE:
    # (Imagine this sends the old messages to Qdrant or a PostgreSQL database)
    await save_to_permanent_database(old_messages)
    
    # 3. DELETE IT FROM RAM:
    # We tell Redis: "Now delete those exact same messages so we free up memory."
    await redis_client.zremrangebyscore(timeline_key, 0, oldest_time)
```

### 🧠 The Production Architecture (Memory Capacity Cascades)
To guarantee stability, Redis strictly holds a rolling cache window. A background cron worker loops through older records, generates dense vector embeddings offline (using Local HuggingFace CPU models like MiniLM), archives them in persistent local disk storage via the Qdrant Vector Database, and evicts the raw text frames from memory using `ZREMRANGEBYSCORE`.

### 🎤 Tech Interview Drill: 8 Questions on Vector Databases

**1. What is a Vector Database and how does it differ from a Relational DB?**
*Answer:* A Relational DB (like Postgres) searches for exact keyword matches. A Vector Database searches by *semantic meaning*. It does this by storing data as mathematical coordinates in a high-dimensional space and measuring the physical distance between them.

**2. What is a Vector Embedding?**
*Answer:* An embedding is an array of floating-point numbers (e.g., `[0.14, -0.99, 2.45...]`) generated by an AI model. These numbers represent the semantic meaning of a word, sentence, or image.

**3. How does a Vector DB measure the similarity between two vectors?**
*Answer:* It uses geometric distance metrics. The most common is **Cosine Similarity**, which measures the angle between two vectors. A smaller angle means the meanings are highly similar.

**4. What is an "Eviction Cascade" in memory management?**
*Answer:* It is the architectural pattern of moving stale or older data out of an expensive, fast storage layer (like Redis RAM) and dumping it into a cheaper, long-term storage layer (like Qdrant/Disk) to free up capacity.

**5. If Qdrant is so smart, why not just store everything in Qdrant from the beginning?**
*Answer:* Writing to disk-based databases takes milliseconds. Writing to Redis RAM takes microseconds. For live, high-frequency audio chunk ingestion (100 times a second), hitting a disk database directly would create a massive bottleneck.

**6. What does `ZREMRANGEBYSCORE` do in Redis?**
*Answer:* It removes all elements in a Sorted Set (ZSET) that have a score within the given minimum and maximum range. It is the perfect command for deleting transcripts older than a specific timestamp limit.

**7. What happens if you try to put a 500-page book into a single Vector?**
*Answer:* The AI model loses context and the vector becomes "diluted" and inaccurate. You must split large texts into smaller "chunks" (like paragraphs) and generate a unique vector for each chunk.

**8. Code Example: How do Vector Databases store the actual human-readable text?**
*Answer:* Vectors are just math. Databases use a `payload` or `metadata` field attached to the Vector ID to store the original text so humans can read the search result.
```python
PointStruct(
    id=123, 
    vector=[0.1, 0.2], 
    payload={"text": "The budget is approved."} # The metadata!
)
```
